title: View a trace
description: Inspect a trace in LangSmith using the Messages view or Details view.
---

When you open a trace in the [LangSmith UI](https://smith.langchain.com), two views are available at the top of the trace panel:

- [**Messages**](#messages-view): Renders a trace as a readable conversation, showing the sequence of messages exchanged between the user, assistant, and any tools called.
- [**Details**](#details-view): Shows the complete technical record of a trace, including every run with its inputs, outputs, timing, token counts, and metadata.

## Messages view

The Messages view is the **default view** for traces under 20 MB.

<Note>
LangSmith does not display the Messages view for traces larger than 20 MB. On [self-hosted](/langsmith/self-hosted) deployments, this limit is configurable via the `MESSAGE_VIEW_MAX_BYTES` environment variable. Set to `0` to disable the limit.
</Note>

Use the Messages view to understand what happened in an agent interaction quickly: what the user asked, how the assistant responded, and which tools the agent invoked.

### Supported integrations

The Messages view appears automatically when LangSmith detects a supported integration. No configuration is required — detection is based on metadata that each SDK sets automatically.

Supported integrations include:

- LangChain
- Anthropic SDK
- Claude Agent SDK / Claude Code
- OpenAI Chat Completions
- OpenAI Responses API / Agents SDK
- Google Gemini
- Vercel AI SDK

If your integration is not listed, the Messages view may still appear if LangSmith can infer the message format from the payload structure. You can also override detection manually using the `ls_message_format` metadata key described in [Customize the Messages view](#customize-the-messages-view).

### What the Messages view shows

Each LLM call in the trace appears as a group of messages. The metadata row above each group shows:

- **Model name**
- **Token usage** (total tokens for the call)
- **Cost** (total cost for the call)
- An **LLM call** link to the corresponding run in the Details view (shown only when the AI message has visible text)

Tool calls appear below the assistant message that triggered them. Each tool call card includes a **Tool invocation** link to its run in the Details view.

When an agent makes the same tool call multiple times in a row (single-tool groups), those calls are coalesced into a collapsible row showing the count and tool name. Expand it to see each individual call. When an agent calls multiple different tools in parallel, those are kept as a grouped block and are not coalesced.

Sub-agent interactions show a preview link to the nested trace.

Collapsed and expanded message state is preserved when you switch between the Messages and Details tabs.

### Provider-specific behavior

The messages shown in the Messages view depend on the integration:

| **Integration** | **Messages shown** |
|-------------|---------------|
| Anthropic / Claude SDK | Messages from the current trace only |
| DeepAgents / OpenAI | Messages from the current trace and ancestor traces |

### Customize the Messages view

Tracing wrapper integrations from the latest versions of the LangSmith SDK set the metadata needed for the Messages view automatically. If you use a supported integration, you typically do not need to configure anything manually.

If you are building custom instrumentation, you can control how runs appear in the Messages view using metadata keys on individual runs.

- `ls_agent_type`: Controls where messages from an agent-like run appear. Accepted values:

  | Value | Messages view behavior |
  |---|---|
  | `"root"` | Messages from this run appear in the main Messages view. |
  | `"subagent"` | Messages from this run appear in a side thread, separate from the main conversation. |
  | `"middleware"` | Messages from this run are hidden from the Messages view. |

  ```python
  @traceable(metadata={"ls_agent_type": "root"})
  def my_agent():
      ...
  ```

- `ls_message_format`: Overrides automatic format detection. Accepted values:
  - `"langchain"` — parse as LangChain message format
  - `"anthropic"` — parse as Anthropic message format
  - `"responses"` — parse as OpenAI Responses API format

- `ls_message_view_exclude`: Set to `true` to exclude a run and all runs in its subtree from the Messages view.

    ```python
    @traceable(metadata={"ls_message_view_exclude": True})
    def my_run():
        ...
    ```

## Details view

Use the Details view to debug execution: inspect a specific LLM call, review tool inputs and outputs, or identify where an error occurred.

### Customize the Details view

Setting `run_type="llm"` on a run causes the Details view to render token counts and latency for that run. See [Log an LLM trace](/langsmith/log-llm-trace) for the full message format specification.

Tool messages are auto-expanded when a run's `run_type` is `tool`.

### Actions

From the Details view, you can also:

- **Compare traces**: Select two traces to open a side-by-side comparison. See [Manage a trace](/langsmith/manage-trace#compare-traces).
- **Share a trace**: Generate a public link to the trace. See [Manage a trace](/langsmith/manage-trace#share-a-trace).
- **View server logs**: Access server logs associated with a trace generated by a LangSmith deployment. See [Manage a trace](/langsmith/manage-trace#view-server-logs).
