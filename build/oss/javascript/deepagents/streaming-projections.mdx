Deep Agents build on LangGraph's Projection Streaming model and add a first-class `subagents` projection. Use it when you want application-facing streams for the coordinator agent, delegated subagents, nested messages, tool calls, and final state.

<Note>
Interested in lower-level Pregel stream modes and the existing Deep Agents streaming guide? See the [Mode Streaming page](/oss/javascript/deepagents/streaming).
</Note>

## Stream subagents

Deep Agents add a subagent projection on top of LangGraph streaming. Use `run.subagents` when you want one stream handle per delegated `task` call.



```ts
const run = await agent.stream_v2({
  messages: [{ role: "user", content: "Write me a haiku about the sea" }],
});

for await (const subagent of run.subagents) {
  console.log(subagent.name);
  console.log(await subagent.taskInput);

  for await (const message of subagent.messages) {
    console.log(await message.text);
  }
}
```


## Subagent stream fields

Each subagent stream is run-shaped, so you can recurse into it the same way you consume the parent run.



TypeScript uses camelCase projection names such as `toolCalls` and `taskInput`.


| Field | Description |
| ----- | ----------- |
| `name` | Subagent name. |
| `taskInput` | Prompt or input passed to the task tool. |
| `path` | Namespace path for the subagent run. |
| `status` | Lifecycle status such as `started`, `running`, `completed`, `failed`, or `interrupted`. |
| `messages` | Messages emitted by the subagent. |
| `subagents` | Nested subagent invocations. |
| `output` | Final subagent state. |

| `toolCalls` | Tool calls scoped to the subagent. |

## Stream messages

Deep Agents can emit messages from the coordinator agent and from delegated subagents. Use `run.messages` for top-level messages and `subagent.messages` for each delegated subagent.



```ts
const run = await agent.stream_v2(input);

for await (const message of run.messages) {
  console.log("[coordinator]", await message.text);
}

for await (const subagent of run.subagents) {
  for await (const message of subagent.messages) {
    console.log(`[${subagent.name}]`, await message.text);
  }
}
```


## Stream tool calls

Deep Agents expose tool calls at each level of the agent tree. Use the top-level `run.tool_calls` for coordinator tools and each `subagent.tool_calls` for delegated work.



```ts
const run = await agent.stream_v2(input);

for await (const call of run.toolCalls) {
  console.log("[coordinator tool]", call.name, call.input);
  console.log(await call.status);
}

for await (const subagent of run.subagents) {
  for await (const call of subagent.toolCalls) {
    console.log(`[${subagent.name} tool]`, call.name, call.input);

    const status = await call.status;
    if (status === "finished") {
      console.log(await call.output);
    } else if (status === "error") {
      console.error(await call.error);
    }
  }
}
```

## Consume concurrently

Coordinator and subagent output often interleave. Consume projections concurrently when you need live UI updates.



Use concurrent consumers in JavaScript:

```ts
const run = await agent.stream_v2(input);

await Promise.all([
  (async () => {
    for await (const message of run.messages) {
      console.log("[coordinator]", await message.text);
    }
  })(),
  (async () => {
    for await (const subagent of run.subagents) {
      void (async () => {
        for await (const message of subagent.messages) {
          console.log(`[${subagent.name}]`, await message.text);
        }
      })();
    }
  })(),
]);
```

## Subagents versus subgraphs

`run.subgraphs` shows graph execution structure. `run.subagents` shows product-level Deep Agents task delegations. Use `run.subagents` for user-facing UI because it hides internal graph nodes and exposes the subagent concept directly.
