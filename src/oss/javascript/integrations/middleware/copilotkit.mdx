You typically implement the [Deep Agent](/oss/deepagents/overview) graph in **Python** and add [`CopilotKitMiddleware`](https://docs.copilotkit.ai/langgraph/deep-agents) to the `create_deep_agent` or `create_agent` call so the graph is AG-UI–compatible. See the [Python CopilotKit integration](/oss/integrations/middleware/copilotkit) for installation, the `create_agent` + FastAPI pattern, and how that maps to CopilotKit features (generative UI, HITL, shared state).

In **TypeScript**, you mount [CopilotKit](https://www.copilotkit.ai/)'s client and server packages so the UI talks to that deployment (local or hosted). The usual pattern is a [Next.js](https://nextjs.org) API route that uses `CopilotRuntime` with a `LangGraphHttpAgent` or `LangGraphAgent` from CopilotKit (see [LangGraph and CopilotKit](https://docs.copilotkit.ai/langgraph)), plus `@copilotkit/react-core` (or `/v2` APIs) in your app shell.

For a long-form LangGraph + CopilotKit + React path (including a Python agent, generative UI, and `CopilotKitMiddleware`), read [CopilotKit](/oss/langchain/frontend/integrations/copilotkit). For a Deep Agents—focused guide from the CopilotKit side, use [Deep Agents and CopilotKit (CopilotKit)](https://docs.copilotkit.ai/langgraph/deep-agents).

## TypeScript: runtime and client

The exact route code depends on whether the browser talks to LangGraph over HTTP or through the LangGraph Platform, but the shape is the same: a single Copilot runtime endpoint, then the React `CopilotKit` provider and chat UI. Follow the "Setup Copilot Runtime" and "CopilotKit Provider" sections in [Deep Agents and CopilotKit (CopilotKit)](https://docs.copilotkit.ai/langgraph/deep-agents) and use the `LangGraphHttpAgent` or `LangGraphAgent` example that matches your deployment.
