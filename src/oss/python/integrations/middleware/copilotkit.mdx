[CopilotKit](https://www.copilotkit.ai/) publishes the Python [`CopilotKitMiddleware`](https://docs.copilotkit.ai) so any LangGraph, LangChain, or deep agent can speak the [Agent UI (AG-UI)](https://docs.ag-ui.com/) wire protocol, stream tool and message events to a chat UI, and read or write the shared **CopilotKit** slice of state. The same package also exposes server helpers to mount a CopilotKit-compatible HTTP endpoint in front of your graph.

**Typical use cases** include:

- **[Generative UI](https://docs.copilotkit.ai/langgraph/generative-ui)**: The model and tools drive structured or component-based UI. Pair `CopilotKitMiddleware` with additional middleware to forward a UI schema (for example from `useAgentContext` in React) into a structured output contract, as in [CopilotKit with LangGraph and React](/oss/langchain/frontend/integrations/copilotkit).
- **[Human in the loop](https://docs.copilotkit.ai/langgraph/human-in-the-loop)**: Use CopilotKit HITL flows to pause, approve, or override agent steps from the app while the graph is running.
- **[Shared state](https://docs.copilotkit.ai/langgraph/shared-state)**: Expose data from your product to the model and keep the app aligned with the agent in real time through the CopilotKit state channel.

<Info>
  For end-to-end HTTP wiring, the CopilotKit React and runtime stack, and UI patterns, see the
  [CopilotKit docs for LangGraph](https://docs.copilotkit.ai/langgraph) and
  [CopilotKit with LangGraph and React](/oss/langchain/frontend/integrations/copilotkit).
</Info>

## What you get on the server

The [copilotkit](https://pypi.org/project/copilotkit/) and related packages give you a bridge between a LangGraph deployment and CopilotKit clients.

| Component | Role |
| --------- | ---- |
| `CopilotKitMiddleware` | Merges CopilotKit/AG-UI state and requests into your agent, including frontend [tool calls](/oss/langchain/agents#tools) and context. Add it to the `middleware` list for @[create_agent] or @[create_deep_agent]. |
| `CopilotKitState` (subclass) | [Custom state](/oss/langchain/agents#memory) (extend `CopilotKitState`) so the CopilotKit key is part of the graph state. |
| `LangGraphAGUIAgent` | Bundles a compiled graph with a name and description for the runtime. |
| `add_langgraph_fastapi_endpoint` (from [ag-ui-langgraph](https://pypi.org/project/ag-ui-langgraph/)) | Wires a **FastAPI** app so CopilotKit can run your graph through the same [LangGraph](/oss/langgraph/overview) deployment. Use it when you add a [custom `http` app in `langgraph.json`](/oss/langchain/frontend/integrations/copilotkit#extend-the-langgraph-deployment-with-a-custom-endpoint) instead of a standalone process. |

You can also keep using **Deep Agents** as your graph: `CopilotKitMiddleware` is the same middleware whether you use @[create_deep_agent] or a graph built with @[create_agent], as long as you add it to the list.

## Setup

### Installation

The middleware lives in the [copilotkit](https://pypi.org/project/copilotkit/) package alongside your Deep Agents stack. Install `deepagents`, `copilotkit`, and your [chat model](/oss/integrations/chat) package (this example uses OpenAI):

<CodeGroup>
```python pip
pip install -U deepagents copilotkit langchain-openai
```

```python uv
uv add deepagents copilotkit langchain-openai
```
</CodeGroup>

**Graph plus FastAPI bridge** (serve CopilotKit over the same app as a LangGraph deployment):

```bash
uv add copilotkit ag-ui-langgraph fastapi uvicorn
```

### API keys and tracing

- Set the API key (or other credentials) for whichever model id you use.
- [LangSmith](/langsmith/home) tracing is optional locally and is commonly used when you [deploy a graph](/oss/langgraph/deploy) to the LangGraph Platform or run the LangGraph dev server.

## Use with a Deep Agent

Add `CopilotKitMiddleware` to the `middleware` list passed to @[create_deep_agent]. The middleware is what lets CopilotKit route frontend-exposed tool calls and align chat state with your graph. Keep any other [middleware you configure](/oss/deepagents/customization#middleware) in the same list.

```python
from deepagents import create_deep_agent
from copilotkit import CopilotKitMiddleware
from langgraph.checkpoint.memory import MemorySaver


def get_weather(location: str) -> str:
    """Return a simple weather string for a location."""
    return f"The weather in {location} is sunny."


agent = create_deep_agent(
    model="openai:gpt-5.4",
    tools=[get_weather],
    middleware=[CopilotKitMiddleware()],  # AG-UI + frontend tools and context
    system_prompt="You are a helpful research assistant.",
    checkpointer=MemorySaver(),
)
```

In this form the **compiled graph** is ready to plug into a **CopilotKit- or AG-UI–aware** process (for example, the FastAPI pattern in the next section) or a guide such as [Deep Agents and CopilotKit (CopilotKit)](https://docs.copilotkit.ai/langgraph/deep-agents).

## Build a graph with create_agent

The same `CopilotKitMiddleware` works with a graph built from @[create_agent] when you declare state that extends `CopilotKitState`: you compile the agent, wrap it in `LangGraphAGUIAgent`, and expose the FastAPI app as the `http` entrypoint in `langgraph.json` so a deployment can serve the graph and the chat endpoint together.

```python
from copilotkit import CopilotKitMiddleware, CopilotKitState, LangGraphAGUIAgent
from ag_ui_langgraph import add_langgraph_fastapi_endpoint
from fastapi import FastAPI
from langchain.agents import create_agent
from langgraph.checkpoint.memory import MemorySaver


class AgentState(CopilotKitState):
    pass


def get_weather(location: str) -> str:
    """Return a simple weather string for a location."""
    return f"The weather in {location} is sunny."


graph = create_agent(
    model="openai:gpt-5.4",
    tools=[get_weather],
    middleware=[CopilotKitMiddleware()],
    state_schema=AgentState,
    system_prompt="You are a helpful assistant.",
    checkpointer=MemorySaver(),
)

app = FastAPI()

add_langgraph_fastapi_endpoint(
    app=app,
    agent=LangGraphAGUIAgent(
        name="my_copilot",
        description="A LangGraph agent for CopilotKit and AG-UI.",
        graph=graph,
    ),
    path="/",
)
```

**Structured generative UI** (for example with `useAgentContext` and an `output_schema` from the client) needs extra middleware that reads Copilot state and applies a [structured output](/oss/langchain/agents#structured-output) strategy, as in the [Python examples](/oss/langchain/frontend/integrations/copilotkit) on the CopilotKit and LangGraph page. Keep that logic next to the components you support in React.

**Deployment**: Mount `app` with the `http` entry in `langgraph.json` and your [LangGraph or LangSmith deployment flow](/oss/langgraph/deploy) so a single process exposes the graph to LangGraph and the same FastAPI app to the CopilotKit client. The [CopilotKit and LangGraph](/oss/langchain/frontend/integrations/copilotkit#extend-the-langgraph-deployment-with-a-custom-endpoint) page shows a concrete `langgraph.json` plus FastAPI example.

## Connect the frontend

- Point your CopilotKit `runtimeUrl` at the route that your FastAPI (or other) app exposes, not always at the raw graph REST surface alone, when you use the [CopilotKit](https://docs.copilotkit.ai) runtime adapter.
- Follow the **TypeScript** examples in the CopilotKit documentation when you wire [LangGraphHttpAgent](https://docs.copilotkit.ai/langgraph) or `LangGraphAgent` in the Node **CopilotRuntime**; the **Python** graph and middleware stay the source of truth for behavior.
